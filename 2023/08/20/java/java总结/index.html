一、其他

1.1 包

​ 1.包的作用？

​ 包就是文件夹，用来管理各种不同功能的java类

​ 2.包名书写规则？

​ 公司域名反写+加包的作用，需要全部英文小写，见名知意。

​ 3.是么是全类名？

​ 报名 + 类名

​ 4.什么时候需要导包，什么时候不需要导包？

​ 使用同一包中的类时，不需要导包

​ 使用java.lang中的类时，不需要

​ 其他情况都需要

​ 如果同时使用两个包中的同类名，需要用全类名

1.2 final

​ 1.final修饰方法：最终方法，不能被重写

​ 2.final修饰类：最终类，不能被继承

​ 3.final修饰变量：是常量，不能被修改

1.3 权限修饰符

1.4 代码块

​ 1.代码块的分类

​ 局部代码块、构造代码块、静态代码块

​ 2.局部代码块的作用

​ 提前结束变量的生命周期（已淘汰）

​ 3.构造代码块的作用

​ 抽取构造构造方法中重复的代码（不够灵活）

​ 4.静态代码块的作用

​ 数据的初始化（重点）

二、正式

抽象类

注意：

  • 抽象类不能实例化
  • 抽象类中不一定有抽象方法，有抽象方法一定是抽象类
  • 可以有构造方法
  • 抽象类的子类
    • 要么重写抽象类中的所有抽象方法
    • 要么是抽象类

​

总结：

​ 1.抽象类的作用是什么?

​ 抽取共性时，无法确定方法体是什么，就把方法定义为抽象的。

​ 强制让子类按照某种格式重写

​ 抽象方法所在的类，必须是抽象类

​ 2.抽象类和抽象方法的格式？

​ publice abstract 返回值类型 方法名(参数列表);

​ publice abstract class 类名{}

​ 3.继承抽象类有哪些要注意？

​ 要么重写抽象类中的所有抽象方法

​ 要么是抽象类

接口

1.1 定义和使用

  • 定义
    • Public interface 接口名 {}
  • 接口不能实例化
  • 接口和类之间是实现关系，通过implements关键字表示
    • Public class 类名 implements 接口名 {}
  • 接口的子类（实现类）
    • 要么重写接口中的所有抽象方法
    • 要是是抽象类

1.2 成员特点

  • 成员变量
    • 只能是常量
    • 默认修饰符：public static final
  • 构造方法
    • 没有
  • 成员方法
    • 只能是抽象方法
    • 默认修饰符：public abstract

1.3 接口的方法

总结：

​ jdk7前：接口只能定义抽象方法

​ jdk8：可以定义有方法体的方法。（默认、静态方法）

​ jdk9：接口中可以定义私有方法

​ 私有方法有两种，普通私有方法，静态私有方法
